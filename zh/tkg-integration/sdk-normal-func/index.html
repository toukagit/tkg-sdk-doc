SDK常用接口

SDK常用接口包含基础部分、广告、事件埋点。

基础部分​

初始化 —— InitSDK（必须调用）​

接口功能：初始化SDK。包含统计、广告SDK初始化，广告预加载，用户协议，实名认证。
调用时机：游戏首个场景加载时调用。(初始化方法只能调用一次)
API说明：
void InitSDK(Action _initCallback = null)

  • 参数：
    _initCallback：初始化结束后回调。可用于处理走完用户协议、实名认证后再继续游戏内容。

示例代码：

TKGSDKManager.Instance.InitSDK();

游戏失去/获得焦点 —— SetGameFocusListener​

接口功能：设置游戏失去/获得焦点时的回调。展示广告时游戏失去焦点，参数为false；关闭广告时，游戏获得焦点，参数为true。播放广告/关闭广告时暂停/恢复游戏声音可在此方法处理。
调用时机：可在调用完初始化方法后立即调用该方法。
API说明：
void SetGameFocusListener(Action<bool> _gameFocusAction)

  • 参数：
    _gameFocusAction：失去焦点，false时回调；获得焦点，true时回调。

示例代码：

TKGSDKManager.Instance.SetGameFocusListener((_isFocus) => {
    if (_isFocus)
    {
        Debug.Log("set game focus");
    }
    else
    {
        Debug.Log("lose game focus");
    }
});

广告部分​

播放插屏 —— PlayInterstitialAd​

接口功能：播放插屏广告。内部根据传入点位名称参数做插屏展示打点。
调用时机：需要播放插屏广告时调用。
API说明：
void PlayInterstitialAd(string _adPos, Action _callback = null, IVADType _IvType = IVADType.IV1)

  • 参数：
    _adPos：触发插屏的点位名称（需求提供该字符串）
    _callback：插屏播放回调，广告关闭、播放失败都会走到这个回调
    _IvType：用于处理不同点位广告频次，默认为点位1，可通过在线参数控制频次。

示例代码：

TKGSDKManager.Instance.PlayInterstitialAd("stage_end", () => {
    NextStageIndex = CurrentStageIndex;
    SceneManager.Instance.Load(SceneDefine.GameScene, true);
});

播放激励视频 —— PlayRewardAd​

接口功能：播放激励视频。内部根据传入参数做激励视频相关打点。
调用时机：需要播放激励视频广告时调用。
API说明：
void PlayRewardAd(string _adPos, Action<bool> _rewardCallback = null, Action _playFailedCallback = null)

  • 参数：
    _adPos：激励视频对应点位名称（需求提供该字符串）
    _rewardCallback：奖励回调，参数为奖励成功/奖励失败
    _playFailedCallback：激励视频播放失败。如调用播放前有暂停游戏，可在这里恢复

示例代码：

TKGSDKManager.Instance.PlayRewardAd("super_warrior", (_isReward) => {
    if (_isReward)
    {
        OnButton_RewardFreeGoldenWarrior();
        justShowVideo = true;
    }
});

插屏是否准备好 —— IsReadyInterstitialAd​

接口功能：判断插屏广告是否准备好。
调用时机：一般无需自行调用。
API说明：
bool IsReadyInterstitialAd()

  • 参数：
    插屏广告准备好，返回true；插屏广告没准备好，返回false。

示例代码：

bool isReadyIV = TKGSDKManager.Instance.IsReadyInterstitialAd();

激励视频是否准备好 —— IsReadyRewardAd​

接口功能：判断激励视频广告是否准备好。
调用时机：一般无需自行调用。
API说明：
bool IsReadyRewardAd()

  • 参数：
    激励视频广告准备好，返回true；激励视频广告没准备好，返回false。

示例代码：

bool isReadyRV = TKGSDKManager.Instance. IsReadyRewardAd();

事件埋点​

普通事件打点接口 —— LogEvent​

接口功能：普通游戏事件打点接口。
调用时机：根据打点需求参数个数调用合适的重载接口。
API说明：
void LogEvent(string _eventSort)
void LogEvent(string _eventSort, string _key, string _value)
LogEvent(string _eventSort, string _key01, string _value01, string _key02, string _value02)
LogEvent(string _eventSort, Dictionary<string, string> _eventDic = null)

  • 参数：
    根据需求中事件和属性个数不同，来方便调用不同重载接口。

示例代码：

TKGSDKManager.Instance.LogEvent("click_shop");
TKGSDKManager.Instance.LogEvent("stage_start", "stage_level", "1");
TKGSDKManager.Instance.LogEvent("stage_time", "stage_level", "1", "time_long", "13");

关卡开始事件 —— NotifyGameStart（关卡类游戏必须调用）​

接口功能：通知关卡开始事件，SDK内部会以此作为关卡开始，来进行关卡开始、结束、时长统计打点。
调用时机：关卡开始时调用。
API说明：
void NotifyGameStart(int _level)
void NotifyGameStart(string _level)

  • 参数：
    关卡ID

示例代码：

TKGSDKManager.Instance.NotifyGameStart(1);

关卡结束事件 —— NotifyGameEnd（关卡类游戏必须调用）​

接口功能：通知关卡结束事件，SDK内部会以此作为关卡结束，来进行关卡结束、时长统计事件上报。
调用时机：关卡结束时调用。
API说明：
void NotifyGameEnd(int _level, bool _win) void NotifyGameEnd(string _level, bool _win)

  • 参数：
    _level：关卡ID _win：关卡成功 true; 关卡失败 false

示例代码：

TKGSDKManager.Instance. NotifyGameEnd(1);

激励视频按钮展示打点 —— LogRewardAdBtnShow​

接口功能：激励视频按钮展示打点封装。
调用时机：激励视频按钮展示时调用。
API说明：
void LogRewardAdBtnShow(string _pos)

  • 参数：
    _pos：激励视频点位

示例代码：

TKGSDKManager.Instance.LogRewardAdBtnShow("double_coin");
